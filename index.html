Tarantool

Key features

  • SQL
  • ACID
  • master-master репликация
  • sharding из коробки
  • Application server + database
  • open-source

Джвижок базы данных

  • memtx (Always in-memory != cache)
  • vinyl (часть данных на диске)

Durability

  • snapshots (начиная с 1.7 системный copy-on-write заменен на самописный механизм наподобие MVCC, за счет этого снапшоты безболезненные)
  • WAL
  • shard redundancy

Последовательный WAL

Дисковые БД vs in-memory

В дисковых БД при записи кроме WAL (последовательная запись ~ миллион обращений в секунду) еще обновляется b-tree индекс (случайное обращение к диску ~ 5000 обращений в секунду). В in-memory нет необходимости читать и писать индексы на диск, они все в памяти.

vinyl tarantool тоже обновляет индексы на диске, но у них другая структура (LSM-tree).

Индексы

  • primary, secondary, multi-part
  • HASH, TREE, BITSET, and RTREE
